git config --global user.name blabla

Дана утиліта 'git config' дозволяє переглядати та налаштовувати параметри, що контролюють усі аспекти роботи Git, а також його зовнішній вигляд. Перше, що слід зробити після встановлення Git - вказати ваше ім'я та адресу електронної пошти. Це важливо, тому що кожен коміт в Git містить цю інформацію, і вона включена в коміти, що передаються вами, і не може бути змінена далі:

git

Дана функція створює порожній Git репозиторій або повторно ініціалізує існуючий

git status

Показує поточний статус робочої вітки

git add

Використовується, щоб почати контролювати новий файл та включити його до усіх комітів

git commit

Використовується для запису зміг до репозиторію

git log

Показує журнали комітів

git checkout

Перемика' гілки або відновлю' робочі файли вітки

git branch

Записує, створює або видаляє гілки

git push

Оновлює віддалені посилання разом із пов’язаними об’єктами

git pull

Дозволяє отримати файл з іншого репозиторію або локальної гілки та інтегрувати з ним

git merge

Дозволяє об’єднати дві або більше вітки проєкту

git clone

Клонує репозиторій в новій директорії

Gitignore файл

Визначає навмисно невідстежувані файли, які потрібно ігнорувати.Файл gitignore визначає навмисно невідстежувані файли, які Git має ігнорувати. Файли, які вже відстежуються Git, не впливають. Кожен рядок у файлі gitignore визначає шаблон. Вирішуючи, чи ігнорувати шлях, Git зазвичай перевіряє шаблони gitignore з кількох джерел у порядку пріоритету, від найвищого до найнижчого.

Advanced

git revert

Дана команда скасовує деякі наявні коміти

git reset

Дозволяє скинути поточну головну гілку до вказаного стану

git rebase

Повторно застосовує коміти поверх іншої базової підказки

git cherry-pick

Застосовує зміни, внесені деякими наявними комітами, записуючи новий коміт для кожного. Це вимагає, щоб ваше робоче дерево було чистим (без змін від коміту HEAD).

git stash

Використовуйте git stash, коли ви хочете записати поточний стан робочого каталогу та індексу, але хочете повернутися до чистого робочого каталогу. Команда зберігає ваші локальні зміни та повертає робочий каталог відповідно до коміту HEAD.
